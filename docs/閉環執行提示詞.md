### Step 1：取得職缺資料
GET https://api-hr.step1ne.com/api/jobs/{job_id}
Headers: Authorization: Bearer {API Key}

提取以下欄位，對應三層審核：

**A 層（硬性條件 — 不符合直接淘汰）：**
- `rejection_criteria` → 明確的淘汰規則
- `exclusion_keywords` → 排除關鍵字

- `education_required` → 最低學歷

**B 層（送人條件 — 不符合仍匯入，標記給顧問）：**
- `submission_criteria` → 顧問的送人偏好條件

**C 層（加分條件）：**
- `talent_profile` → 理想人才描述，其中「加分」項目為 C 層

### Step 2：建立爬蟲任務（含自動換關鍵字機制）

#### 關鍵字策略
根據職缺自行分析生成搜尋關鍵字，準備 3 組備用策略：

- **第一組（精準）**：核心技術 + 職稱，例如 `Java Spring Boot Backend Engineer Taiwan`
- **第二組（廣泛）**：放寬職稱，只保留核心技術，例如 `Java Spring Boot Taiwan`
- **第三組（最廣）**：只用最關鍵的 1-2 個技術 + 地區，例如 `Java Developer Taiwan`

#### 自動換關鍵字規則
執行完 Step 4 後，如果符合以下任一條件，**自動用下一組關鍵字重新執行 Step 2-4**：
- 搜尋結果 < 10 人
- A 層通過率 < 20%（通過人數 / 總搜尋人數）
- 重複換關鍵字最多 3 次，第 3 次用第三組策略後無論如何都繼續往下

在回報結果（Step 9）中，需說明：
- 共嘗試幾組關鍵字
- 每組的搜尋結果數與 A 層通過數
- 最終採用哪一組

### Step 5：三層審核（必須依序 A → B → C）

#### A 層 — 硬性條件（用搜尋結果的 snippet 資訊初步篩選）
逐一檢查每個候選人，對照 `rejection_criteria`、`exclusion_keywords`、`experience_required`、`education_required`、`title_variants`、`key_skills`：

- 職稱是否與 `title_variants` 相關？不相關 = 淘汰
- 技術是否與 `key_skills` 有交集？完全無關 = 淘汰
- 是否命中 `exclusion_keywords`？命中 = 淘汰
- 是否符合 `rejection_criteria` 中的任一條？符合 = 淘汰
- 年資是否 ≥ `experience_required`？不足 = 淘汰
- 學歷是否 ≥ `education_required`？不足 = 淘汰

**原則：資訊不足無法判斷的項目，一律通過，交給 B 層處理。**

→ A 層不通過 = REJECT，不進入 B 層，不匯入系統。

#### B/C 層 — LinkedIn 深度審核 + 履歷下載（用 Chrome 瀏覽器操作）

**只對 A 層通過的候選人執行。** 逐一開啟 LinkedIn profile 頁面，下載 PDF 履歷並讀取完整資料。

##### 操作流程（每個候選人）

1. **開新分頁**：用 `tabs_create_mcp` 建立新分頁
2. **前往 LinkedIn**：用 `navigate` 前往候選人的 LinkedIn URL
3. **等待頁面載入**：用 `computer` 的 `wait` 動作等待 3 秒
4. **下載 LinkedIn PDF 履歷**（核心步驟）：
   a. 找到 profile 頁面上方的「More」（更多）按鈕，用 `find` 搜尋 "More" 或 "更多" 按鈕
   b. 點擊「More」按鈕展開下拉選單
   c. 等待 1 秒讓選單出現
   d. 找到並點擊「Save to PDF」選項
   e. 等待 3-5 秒讓 PDF 下載完成
   f. PDF 會自動下載到瀏覽器預設下載目錄（通常是 ~/Downloads/）
   g. 記錄下載的檔案名稱（通常格式為 `Profile.pdf` 或 `姓名.pdf`）
   h. **如果找不到 Save to PDF 按鈕**（例如非連結的 profile），跳過 PDF 下載，改用頁面文字資料
5. **讀取頁面內容**：用 `get_page_text` 讀取完整 profile 作為補充資料
6. **提取關鍵資訊**（從頁面文字 + PDF 解析結果綜合判斷）：
   - 工作經歷：每段的職稱、公司名、起迄時間、在職時長
   - 教育背景：學校、學位、科系
   - 技能清單
   - 目前職稱與公司
7. **執行 B 層檢查**：對照 `submission_criteria` 逐條檢查
   - 記錄 `met`: ["符合的條件"]
   - 記錄 `not_met`: ["不符合的條件 + 具體原因"]
   - **B 層不是淘汰機制，不論結果如何全部進入 C 層**
8. **執行 C 層檢查**：對照 `talent_profile` 中的加分項目
   - 記錄 `bonus`: ["具備的加分項"]
9. **關閉分頁**：用 `tabs_close_mcp` 關閉分頁
10. **等待間隔**：等待 30-60 秒再處理下一個人（模擬正常瀏覽行為）

##### 注意事項
- 如果 LinkedIn 頁面要求登入或無法載入，跳過該候選人，標記「無法讀取 LinkedIn」
- 如果「Save to PDF」不可用（非一度連結或 LinkedIn 限制），跳過 PDF 下載，用頁面文字資料
- 不要連續快速開啟多個 LinkedIn 頁面，每人之間至少間隔 30 秒
- 如果遇到 LinkedIn 限制（「你搜尋太頻繁」），立即停止，等待 5 分鐘後再繼續
- 優先用 `get_page_text` 取得文字內容，比截圖更快更準確

### Step 6：評級
- A 層不通過 → **REJECT**（不匯入）
- A 層通過 + B 層全符合 + C 有加分 → **Grade A+**
- A 層通過 + B 層全符合 → **Grade A**
- A 層通過 + B 層部分符合 → **Grade B**
- A 層通過 + B 層大部分不符合 → **Grade C**（仍匯入，讓顧問決定）

- C 層有哪些加分項

### Step 8：上傳 PDF 履歷並解析（核心步驟）

**匯入完成後，對每個有下載到 PDF 的候選人執行：**

POST https://api-hr.step1ne.com/api/candidates/{candidate_id}/resume-parse
Headers: Authorization: Bearer {API Key}
Content-Type: multipart/form-data

```
file: <下載的 LinkedIn PDF 檔案>
format: linkedin
uploaded_by: AI-AutoLoop
```

此 API 會一步完成：
1. 解析 PDF 取得完整結構化資料（工作經歷、教育背景、技能、年資等）
2. 自動更新人選卡片所有核心匹配欄位
3. 將 PDF 附件存到人選卡片

**注意**：
- `candidate_id` 從 Step 7 匯入回傳的 `created` 或 `updated` 陣列中取得
- 如果候選人沒有下載到 PDF，跳過此步驟（該候選人僅使用 Step 7 匯入的基本資料）
- 此步驟會覆蓋 Step 7 匯入的 work_experience 和 education_background，因為 PDF 解析的資料更完整準確

## 重要原則
1. **全程自主**，不問人類任何問題（關鍵字、策略、篩選標準都自己決定）
2. **三層必須依序** A → B → C，不能跳層
3. **A 層寧鬆勿嚴** — 資訊不足就通過
4. **B 層不淘汰** — 只標記，全部往下走
5. **每個匯入的人都要帶完整的 match_summary + work_experience + education_background**
6. 所有審核依據只看職缺卡片欄位，不做額外假設
7. **LinkedIn 操作要像真人** — 每人間隔 30-60 秒，遇到限制立即暫停
8. **PDF 履歷是核心資料來源** — 優先用 PDF 解析結果，頁面文字作為補充
