# Step1ne 閉環系統 — Token 消耗與成本分析

> 更新日期：2026-03-25

## 一、單次閉環 Token 消耗（單一職缺）

| 階段 | Input Tokens | Output Tokens | 說明 |
|---|---|---|---|
| 系統載入（MEMORY.md + 提示） | 20K | 1K | 每次新 session 必要 |
| Phase A（爬蟲 + 篩選 + 匯入） | 30K | 5K | API 呼叫、結果解析、A 層篩選 |
| Phase B（CDP PDF 下載） | 15K | 3K | Playwright 腳本執行 + 監控 |
| Phase B.5（驗證 + 補齊） | 10K | 5K | PDF 文字提取 + PATCH |
| Phase C（AI 分析生成） | 20K | 10K | 11 份分析 JSON 生成 |
| 通知（TG notify） | 5K | 2K | notify_consultant.py |
| **單職缺合計** | **~100K** | **~26K** | **~126K tokens** |

## 四、每次觸發的固定成本

每次從 TG Bot 觸發 `claude -p` 都是**新 session**：

| 項目 | Token | 費用 (Sonnet) | 可避免？ |
|---|---|---|---|
| 系統提示載入 | ~8K | $0.024 | 否（CLI 限制） |
| MEMORY.md 讀取 | ~5K | $0.015 | 否 |
| CLAUDE.md 讀取 | ~3K | $0.009 | 否 |
| 工具定義 | ~4K | $0.012 | 否 |
| **固定成本** | **~20K input** | **~$0.06** | — |

> 如果用 Opus 則固定成本為 ~$0.30/次

## 六、成本優化建議

### 已實施
- [x] TG Bot 閉環預設用 Sonnet（省 80%）
- [x] 本地腳本不經 Claude（爬蟲、PDF、通知）

### 可進一步優化
| 方案 | 預估節省 | 難度 | 說明 |
|---|---|---|---|
| Phase A/B 改成純腳本（不用 Claude） | 省 ~50% | 中 | daily_closed_loop.py 已有完整邏輯 |
| 只讓 Claude 做 Phase C（AI 分析） | 省 ~60% | 中 | A/B/通知全腳本化 |
| Phase C 用 Haiku | 再省 ~75% | 低 | 但分析品質可能降低 |
| 批次分析（多人一次 prompt） | 省 ~30% | 低 | 減少 context 重複載入 |

### 最佳化後的理想成本

| 方案 | 單職缺 | 月成本（22職缺×22天） |
|---|---|---|
| 目前（全 Sonnet） | $0.69 | $334 |
| A/B 腳本化 + C 用 Sonnet | $0.30 | $145 |
| A/B 腳本化 + C 用 Haiku | $0.08 | $39 |
