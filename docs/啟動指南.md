## 方式 A：手動觸發（跟龍蝦說）

### 單一職缺

直接跟龍蝦說：

```
對職缺 #52 執行閉環
```

龍蝦會自動讀取 `docs/閉環執行提示詞.md`，依序執行 Step 1 ~ Step 9。

### 所有招募中職缺

```
對所有招募中的職缺執行閉環
```

龍蝦會先 `GET /api/jobs` 篩選 `job_status = "招募中"`，然後逐一執行。

### 指定多個職缺

```
對職缺 #52, #53, #174 執行閉環
```

### 透過執行長

跟執行長說：

```
幫我讓龍蝦對所有招募中職缺跑閉環
```

執行長會透過 Notifications API 發指令給龍蝦。

## 方式 B：自動排程（Cron + 腳本）

### 前置條件

1. **後端 API 正常運行**（`https://api-hr.step1ne.com` 或本機 `localhost:3001`）
2. **爬蟲系統正常運行**（`https://crawler.step1ne.com`）
3. **Chrome CDP 模式已啟動**（port 9222），且已登入 LinkedIn
4. **環境變數已設定**：
   ```bash
   export API_SECRET_KEY="your-api-key-here"
   export API_BASE="https://api-hr.step1ne.com"       # 或 http://localhost:3001
   export CRAWLER_BASE="https://crawler.step1ne.com"   # 或 http://localhost:5050
   ```

### 安裝

```bash
cd /path/to/headhunter-crawler
python3 -m venv venv
source venv/bin/activate
pip install playwright requests
```

### 手動執行一次（測試）

```bash
cd /path/to/headhunter-crawler
source venv/bin/activate
python scripts/daily_closed_loop.py
```

### 設定 Cron 排程

```bash
crontab -e
```

加入以下內容（每天早上 9 點執行）：

```cron
# Step1ne 每日閉環排程
0 9 * * * cd /path/to/headhunter-crawler && source venv/bin/activate && API_SECRET_KEY="your-key" python scripts/daily_closed_loop.py >> logs/cron.log 2>&1

# Chrome CDP 確保啟動（每天 8:55 啟動）
55 8 * * * /Applications/Google\ Chrome.app/Contents/MacOS/Google\ Chrome --remote-debugging-port=9222 --user-data-dir=/tmp/chrome-cdp --headless=new &
```

### 日誌

```bash
# 查看今天的執行日誌
tail -f logs/cron.log

# 查看歷史記錄
ls logs/
```

## 啟動前檢查清單

| # | 項目 | 檢查方式 | 預期結果 |
|---|------|---------|---------|
| 1 | 後端 API | `curl https://api-hr.step1ne.com/api/health` | `{"status":"ok"}` |

## 常見問題

### 閉環跑到一半中斷了
腳本有 checkpoint 機制，重新執行會自動從上次中斷的位置繼續。

### LinkedIn 被限制了
腳本偵測到限制後會自動暫停 5 分鐘。如果持續被限制，今天的閉環會暫停，明天自動繼續。

### 後端 API 掛了（530）
PDF 會存在本機 `resumes/linkedin_pdfs/`，後端恢復後用批量上傳腳本處理：
```bash
python scripts/batch_upload_pdfs.py
```

### 多台電腦同時跑
腳本有鎖定機制（`system-config/crawl_lock_{job_id}`），同一職缺同一天只會被一台電腦處理。
